## 🎓 学生/学术场景

### 📝 论文润色与降重专家

```
你是一位拥有 10 年经验的学术写作专家，专门负责中文学术论文的润色、降重和逻辑优化。

**任务：**
请对以下论文段落进行深度优化，要求：
1. 保持原意不变，但表达更加学术化和精准
2. 降低与现有文献的重复率（目标：重复率 < 10%）
3. 优化逻辑结构，确保论证链条清晰
4. 检查并修正语法错误、标点符号使用不当等问题

**原文：**
[在此粘贴你的论文段落]

**输出格式：**
- 优化后的文本（直接输出，无需标注）
- 修改说明（列出 3-5 个关键改动点）
- 降重建议（如果重复率仍然较高，提供替换词汇建议）
```

**💡 使用场景：**
- 毕业论文降重（从 40% 降到 10% 以下）
- 期刊投稿前的语言润色
- 英文论文的中文摘要优化

## 💼 职场/求职场景

### 🎯 简历优化与面试官模拟

```
你是一位资深 HR 和职业规划师，拥有 15 年招聘经验，擅长帮助求职者优化简历并通过面试。

**角色设定：**
- 你熟悉互联网、金融、制造业等 10+ 行业的招聘标准
- 你能够识别简历中的"无效信息"并转化为"亮点"
- 你擅长模拟压力面试，帮助候选人提前适应真实场景

**任务一：简历优化**
请分析以下简历内容，并提供：
1. **核心问题诊断**（列出 3 个最严重的问题）
2. **优化建议**（针对每个问题给出具体修改方案）
3. **亮点提炼**（找出 2-3 个可以放大的优势）
4. **优化后的简历片段**（直接输出修改后的版本）

**我的简历：**
[在此粘贴简历内容]

**目标岗位：**
[填写目标岗位名称，如：产品经理、数据分析师等]

---

**任务二：面试官模拟**
现在请你扮演目标公司的面试官，对我进行一轮完整的面试：

1. **自我介绍环节**（1 分钟）
2. **项目经历深挖**（针对简历中的核心项目，连续追问 3-5 个问题）
3. **压力测试**（提出 1-2 个刁难性问题，如："你为什么从上家公司离职？"）
4. **面试反馈**（结束后给出：优点、不足、改进建议）

**开始面试：**
[回复"开始"后，面试官会立即开始提问]
```

**💡 使用场景：**
- 简历投递前最后优化（提升通过率 50%+）
- 大厂面试前的模拟训练（减少紧张感）
- 转行求职时的岗位匹配度分析

## 🔧 指令调优小技巧

> **核心公式：** 好指令 = 角色设定 + 任务描述 + 格式要求 + 示例输出

### 📋 三要素拆解

#### 1️⃣ 角色设定（Who）

**❌ 错误示例：**
```
帮我写一篇文章
```

**✅ 正确示例：**
```
你是一位拥有 10 年经验的 [行业] 专家，擅长 [具体技能]。
```

**💡 为什么有效？**
- 给 AI 一个明确的"身份"，它会调用对应的知识库
- 越具体的角色，输出质量越高（如："资深产品经理" > "产品经理" > "懂产品的人"）

---

#### 2️⃣ 任务描述（What + How）

**❌ 错误示例：**
```
优化这段文字
```

**✅ 正确示例：**
```
请对以下文字进行优化，要求：
1. 保持原意不变，但表达更加 [具体目标]
2. [具体操作 1]
3. [具体操作 2]
```

**💡 为什么有效？**
- 明确告诉 AI "做什么"和"怎么做"
- 分步骤描述，AI 更容易理解你的意图
- 避免模糊词汇（如："更好"、"优化"），改用具体指标（如："重复率 < 10%"）

---

#### 3️⃣ 格式要求（Output Format）

**❌ 错误示例：**
```
给我一个方案
```

**✅ 正确示例：**
```
**输出格式：**
- 核心问题诊断（列出 3 个）
- 优化建议（针对每个问题给出具体方案）
- 优化后的文本（直接输出）
```

**💡 为什么有效？**
- 结构化输出，方便你直接使用
- 减少后续整理时间
- 确保 AI 不会遗漏关键信息

### 🎯 进阶技巧

#### 技巧 1：使用"思维链"（Chain of Thought）

```
请按以下步骤思考：
1. 首先分析 [问题 A]
2. 然后考虑 [问题 B]
3. 最后综合 [问题 A] 和 [问题 B] 得出结论
```

**适用场景：** 复杂逻辑推理、多因素决策

---

#### 技巧 2：提供"示例输出"（Few-shot Learning）

```
**示例：**
输入：[示例输入]
输出：[示例输出]

**现在请处理：**
输入：[你的实际输入]
```

**适用场景：** 需要特定格式、风格要求严格

---

#### 技巧 3：设置"约束条件"

```
**约束条件：**
- 字数限制：500 字以内
- 风格要求：口语化、接地气
- 禁止使用：专业术语、复杂句式
```

**适用场景：** 内容长度、风格、受众有明确要求

---

### 🚫 常见错误避坑

| ❌ 错误做法 | ✅ 正确做法 |
| :--- | :--- |
| "帮我写得好一点" | "请优化表达，使用更学术化的词汇，避免口语化" |
| "分析一下" | "请从安全性、性能、可维护性三个维度分析代码" |
| "给我几个方案" | "请提供 3 个方案，每个方案包含：成本、时间、风险三个指标" |
